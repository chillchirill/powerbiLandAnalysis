# Структура експорту Land Matrix для Power BI

## Загальні примітки

Цей експорт складається з кількох пов’язаних CSV-файлів. Усі файли мають такі технічні особливості:

- роздільник полів: `;`
- кодування: `UTF-8` або `UTF-8 with BOM`
- текстові поля можуть містити переноси рядків усередині лапок

Для Power BI це означає:

- імпорт треба робити саме як CSV із роздільником `;`
- не можна ділити файл на рядки вручну до парсингу
- пусті рядки краще перетворювати на `null`

## Загальна логіка набору даних

- `deals.csv` — головна таблиця, один рядок = одна земельна угода
- `locations.csv` — географічні точки / місця, пов’язані з угодою
- `contracts.csv` — контракти, пов’язані з угодою
- `datasources.csv` — джерела інформації, на підставі яких описана угода
- `investors.csv` — довідник усіх інвесторів
- `involvements.csv` — зв’язки між інвесторами, наприклад материнська компанія -> дочірня

## Ключі та зв’язки

- `deals[Deal ID]` — головний ключ угоди
- `contracts[Deal ID]` -> `deals[Deal ID]`
- `datasources[Deal ID]` -> `deals[Deal ID]`
- `locations[Deal ID]` -> `deals[Deal ID]`
- `deals[Operating company: Investor ID]` -> `investors[Investor ID]`
- `involvements[Investor ID Downstream]` -> `investors[Investor ID]`
- `involvements[Investor ID Upstream]` -> `investors[Investor ID]`

Важливо:

- у `contracts.csv`, `datasources.csv`, `locations.csv` поле `ID` не є глобально унікальним
- для цих таблиць краще використовувати складений ключ `ID + Deal ID`

## `deals.csv`

### Що це за таблиця

Це головна таблиця всього експорту. Вона описує саму земельну угоду: розмір, країну, статус, інвестора, вплив на громади, використання землі, виробництво, воду, зайнятість та інші характеристики.

### Рівень деталізації

Один рядок = одна угода.

### Ключ

- основний ключ: `Deal ID`

### Важлива примітка для Power BI

Частина колонок у `deals.csv` не є “простими” значеннями. У них закодовано:

- історію змін
- кілька значень в одному полі
- структуру на кшталт `дата#статус#значення`
- кілька записів в одному полі через `|`

Тобто частину колонок цієї таблиці потрібно не тільки завантажити, а й окремо парсити.

## `locations.csv`

### Що це за таблиця

Ця таблиця зберігає географічні локації, які належать до конкретної угоди. В однієї угоди може бути кілька місць.

### Рівень деталізації

Один рядок = одна локація, пов’язана з угодою.

### Ключі

- технічний ідентифікатор: `ID`
- зв’язок із головною таблицею: `Deal ID`
- для Power BI краще використовувати складений ключ: `ID + Deal ID`

### Колонки

| Колонка | Що означає / що містить |
|---|---|
| `ID` | Технічний ідентифікатор локації. Сам по собі не завжди унікальний у всьому експорті. |
| `Deal ID` | Ідентифікатор угоди, до якої належить локація. |
| `Spatial accuracy level` | Рівень просторової точності: точні координати, приблизне місце, адміністративний район тощо. |
| `Location` | Назва місця або текстова географічна прив’язка. |
| `Point` | Координати точки у форматі `latitude,longitude`. Для Power BI краще розділити на два окремих поля. |
| `Facility name` | Назва об’єкта, підприємства, майданчика або виробничої одиниці в цій локації. |
| `Location description` | Детальний текстовий опис місця. |
| `Comment on location` | Коментар або уточнення до локації. |

## `contracts.csv`

### Що це за таблиця

Це таблиця контрактів, що стосуються угоди. В однієї угоди може бути кілька контрактів або кілька етапів договорів.

### Рівень деталізації

Один рядок = один контракт або одна контрактна подія.

### Ключі

- технічний ідентифікатор: `ID`
- зв’язок із головною таблицею: `Deal ID`
- для Power BI краще використовувати складений ключ: `ID + Deal ID`

### Колонки

| Колонка | Що означає / що містить |
|---|---|
| `ID` | Технічний ідентифікатор контрактного запису. |
| `Deal ID` | Ідентифікатор угоди, до якої належить контракт. |
| `Contract number` | Номер контракту або офіційний реєстраційний номер документа. |
| `Contract date` | Дата укладення контракту. |
| `Contract expiration date` | Дата завершення або закінчення строку дії контракту. |
| `Duration of the agreement` | Тривалість договору, як правило в роках. |
| `Comment on contract` | Коментар до контракту, наприклад примітки про юридичні деталі чи етапи погодження. |

## `datasources.csv`

### Що це за таблиця

Тут зібрані джерела даних, на підставі яких описано угоду: статті, урядові документи, звіти, PDF-файли, контактні дані тощо.

Один рядок = одне джерело інформації для конкретної угоди.

### Ключі

- технічний ідентифікатор: `ID`
- зв’язок із головною таблицею: `Deal ID`
- для Power BI краще використовувати складений ключ: `ID + Deal ID`

### Колонки

| Колонка | Що означає / що містить |
|---|---|
| `ID` | Технічний ідентифікатор джерела даних. |
| `Deal ID` | Ідентифікатор угоди, до якої відноситься джерело. |
| `Data source type` | Тип джерела: ЗМІ, урядовий документ, наукова стаття, звіт тощо. |

| `Date` | Дата джерела, публікації або документа. |

## `investors.csv`

### Що це за таблиця

Це довідник усіх інвесторів, які трапляються в базі. Тут зберігається базова інформація про компанії, державні структури, фонди та інші організації.

### Рівень деталізації

Один рядок = один інвестор.

### Ключ

- основний ключ: `Investor ID`

### Колонки

| Колонка | Що означає / що містить |
|---|---|
| `Investor ID` | Унікальний ідентифікатор інвестора. |
| `Name` | Назва інвестора або організації. |
| `Country of registration/origin` | Країна реєстрації або країна походження інвестора. |
| `Classification` | Категорія інвестора: державна компанія, приватна компанія, фонд, публічна компанія тощо. |
| `Investor homepage` | Офіційний сайт інвестора. |
| `Opencorporates link` | Посилання на компанію в OpenCorporates. |
| `Comment` | Додатковий коментар до інвестора. |
| `Action comment` | Службовий або додатковий коментар щодо дій / процесів; у поточному експорті поле виглядає порожнім. |

## `involvements.csv`

### Що це за таблиця

Ця таблиця описує зв’язки між інвесторами. Вона потрібна для відтворення корпоративної структури: хто кому належить, хто материнська компанія, хто дочірня, де є частка власності або кредит.

### Рівень деталізації

Один рядок = один зв’язок між двома інвесторами.

### Ключ

- основний ключ: `Involvement ID`

### Колонки

| Колонка | Що означає / що містить |
|---|---|
| `Involvement ID` | Унікальний ідентифікатор зв’язку між інвесторами. |
| `Investor ID Downstream` | Ідентифікатор “нижчої” компанії в ланцюгу, тобто дочірньої або залежної сторони. |
| `Investor Name Downstream` | Назва дочірньої / залежної сторони. |
| `Investor ID Upstream` | Ідентифікатор “вищої” компанії в ланцюгу, тобто материнської або контролюючої сторони. |
| `Investor Name Upstream` | Назва материнської / контролюючої сторони. |
| `Relation type` | Тип відносин між сторонами, наприклад материнська компанія. |
| `Investment type` | Тип інвестиційного зв’язку: частка капіталу, позика тощо. |
| `Ownership share` | Частка власності у відсотках або частках. |
| `Loan amount` | Сума кредиту або позики, якщо зв’язок фінансовий. |
| `Loan currency` | Валюта позики. |
| `Loan date` | Дата позики або фінансової події. |
| `Comment` | Коментар до зв’язку між інвесторами. |

- багатозначні поля

- `Top parent companies`
- `Negotiation status`

- `Intention of investment`
